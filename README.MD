# Binance historical storage
## Цель проекта
Предпологается, что будет использоваться как API для построения сервисов связанных с выводом информации о криптовалюте (пока только свечи (klines) BTCUSDT). 

## Зависимости от third-party
Как хранилище данных используется Postgres.

Библиотека Libpqxx для взаимодействия с Postgres.

Библиотека flatbuffers для zero-copy сериализации данных и передачи сырого массива байтов в другие сервисы.

Библиотека http-lib для сетевого взаимодействия. В будущем добавлю boost::beast.

Библиотека binacpp для получения данных с биржи Binance. Ее пришлось форкнуть, так как нужна было добавить поддержку api v3 from Binance и CMake сборку.

Все используемые библиотеки затянуты в проект при помощи git submodule.

## Как собрать проект?
Для сборки проекта используется CMake и генератор Ninja. А так же должен быть установлен curl и docker-compose в системе. Проект собирается как STATIC библиотека.

Файл для docker-compose.yml находится в корне репозитория. Подтягивает образы Postgres и adminer (web-морда для Postgres).

Запуск рабочего окружения docker-compose в корне репозитория:
```shell
    sudo docker-compose up -d 
```

Например, чтобы установить curl в Arch Linux, введите команду в терминале:
```shell 
    pacman -Sy curl 
```

Клонируем репозиторий:
```shell
    git clone https://github.com/lpdgrl/FHC.git
```

Выполняем сборку в корне проекта:
```shell
    mkdir build && cd build
    cmake .. -G Ninja
    cmake --build . 
```

Перед запуском примера своего приложения нужно подготовить конфигурацию конфиг-файла (.nfile) в директории config. С репозиторием поставляется шаблонный конфиг-файл .nfile-template и для быстрого старта 
вы можете его переименовать в .nfile

Структура конфиг-файла относительно простая:
```shell
    DB_HOST=localhost    # хост базы данных Postgres
    DB_NAME=postgres     # имя базы данных
    DB_USER=postgres     # имя пользователя базы данных
    DB_PASSWORD=postgres # пароль базы данных
    HTTP_HOST=0.0.0.0    # ip-address http-сервера
    HTTP_PORT=8000       # порт http-сервера
```

Теперь все готово, и чтобы запустить http-пример, введите в терминал:
```shell
    {подставить свой путь}/build/examples/http_example
```
или если находитесь в ./build/examples/

```shell
    ./http-example
```

## Как использовать?
При первоначальном запуске база данных будет заполняться свечами BTCUSDT с хардкоженной даты 2025-08-01 по текущую дату. В будущем поправлю, чтобы можно было с произвольной даты заполнять.

В браузере зайдите на localhost и вам отобразится веб-страница с выбором дат для получения flatbuffer со свечами по BTCUSDT за выбранный период. Для проверки, что массив байтов пришел, необходимо посмотреть в инструментах разработчика->console количество переданных байтов и, что пришел массив сырых байтов. 

Чтобы заполнить базу данных новыми свечами введите в браузере
```shell
    http://localhost:8000/api/update 
```
И после заполнение на странице отобразится количество добавленных свечей в базу данных либо что данные в базе данных актуальны.

## Какой функционал реализован?
SQL Loader для загрузки sql-файлов и подстановка в них параметров при помощи механизма libpqxx.

Передача произвольного GET-обработчика внутрь httplib. Это подробно показано в примере examples\http-example.

Получение и обновление klines по одной криптовалюте.
